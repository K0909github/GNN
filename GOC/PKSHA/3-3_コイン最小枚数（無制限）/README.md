# コイン最小枚数（無制限）

- 出典: PKSHA
- 問題ID: 3-3
- レベル: C
- 目安時間: 30〜40分
- タグ: 動的計画法, コイン問題

## 問題文
N 種類のコインがあり、i 番目のコインの価値は cᵢ 円です。金額 M 円をちょうど支払うために必要なコインの最小枚数を出力してください。支払いが不可能な場合は -1 を出力してください（各コインは無制限に使えます）。

## 制約
- 1 ≤ N ≤ 10³
- 1 ≤ M ≤ 10⁴
- 1 ≤ cᵢ ≤ 10⁴

## 入出力例
### 例1
```text
入力:
3 11
1 5 6

出力:
2
```
注釈: 5+6=11 で 2 枚

### 例2
```text
入力:
2 3
2 4

出力:
-1
```
注釈: 2 と 4 では 3 を作れない

### 例3
```text
入力:
1 10000
1

出力:
10000
```
注釈: 1 円玉 10000 枚

## ヒント
dp[j] = 金額 j を作る最小コイン枚数 と定義します。dp[0] = 0 から始めてみましょう。

## 解説
完全ナップサック問題（無制限コイン）のDPです。

【Python 例】
N, M = map(int, input().split())
C = list(map(int, input().split()))
INF = float('inf')
dp = [INF] * (M + 1)
dp[0] = 0
for j in range(1, M + 1):
    for c in C:
        if j >= c and dp[j - c] != INF:
            dp[j] = min(dp[j], dp[j - c] + 1)
print(dp[M] if dp[M] != INF else -1)

計算量: O(N×M)
※ 遷移: 金額 j を作るには「j-c を作って c を1枚足す」
