# 区間スケジューリング

- 出典: PKSHA
- 問題ID: 2-2
- レベル: B
- 目安時間: 25〜35分
- タグ: 貪欲法, 区間

## 問題文
N 個のタスクがあり、i 番目のタスクは時刻 sᵢ に開始し時刻 eᵢ に終了します（同じ時刻に終了・開始するタスクは重複なく連続して実行できます）。1 人の作業者が重複なく実行できるタスクの最大個数を出力してください。

## 制約
- 1 ≤ N ≤ 2×10⁵
- 0 ≤ sᵢ < eᵢ ≤ 10⁹

## 入出力例
### 例1
```text
入力:
4
1 3
2 5
3 9
6 8

出力:
3
```
注釈: [1,3], [3,9] は重複なので [1,3],[6,8] or [1,3],[3,9] → 3個

### 例2
```text
入力:
3
1 10
2 3
4 5

出力:
2
```
注釈: [2,3] と [4,5] の2つ

### 例3
```text
入力:
1
0 1

出力:
1
```

## ヒント
終了時刻が早いタスクから貪欲に選ぶのが最適解です。

## 解説
終了時刻でソートして貪欲に選択します。

【Python 例】
N = int(input())
tasks = [tuple(map(int, input().split())) for _ in range(N)]
tasks.sort(key=lambda x: x[1])  # 終了時刻でソート
count = 0
last_end = -1
for s, e in tasks:
    if s >= last_end:
        count += 1
        last_end = e
print(count)

計算量: O(N log N)
※ 証明: 終了が早いタスクを選ぶと「残り時間」が最大化される
